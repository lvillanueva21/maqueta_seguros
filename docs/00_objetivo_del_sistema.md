# Objetivo del sistema

BROKER SEGUROS digitaliza progresivamente expedientes y procesos de seguros sin obligar flujos artificiales.

## Modelo vigente

- Contacto de gestión: persona natural interesada o relacionada con el proceso.
- Cliente: empresa o consorcio. Puede estar pendiente al crear expediente.
- Empresa: se registra con razón social o RUC; el RUC, cuando exista, es válido y único.
- Consorcio: tiene al menos dos empresas; puede usar RUC propio o el RUC de una empresa operadora tributaria.
- Expediente: requiere contacto, nombre y descripción; cliente, cotización y póliza se completan después.

La etapa actual es una maqueta sin MySQL para validar interfaces y reglas.
